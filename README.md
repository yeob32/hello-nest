## Express on NestJS

- express 를 직접 접근하지 않도록 한다.
  - node 프레임워크 전환이 어려워진다. (ex Koa, Fastify)

```
@Get()
some(@Req() req, @Res() res) { }
```

## validate

### class-validator, class-transformer

- 전역 파이프 설정 시 웹 요청에 대한 유효성 검사 편리 제공

```
{
  "statusCode":400,
  "property hacked should not exist",
  "message":["name must be a string",
  "email must be a string"],
  "error":"Bad Request"
}
```

### @nestjs/mapped-types

- 확장한 객체의 프로퍼티를 nullable 하게 사용할 수 있도록 도와줌

## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ npm install
```

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
